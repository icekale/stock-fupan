日期：2026-05-26

## 1. 重要性

该 HTML 不是普通视觉参考，而是本项目最终报告形态的核心目标。项目的价值不只是生成一段复盘文字，而是稳定产出一份接近交易员复盘长文/长图的结构化报告：有判断、有验证、有板块逻辑、有持续性排序、有明日计划、有回避清单。

后续所有 ReportDTO、LLM 输出结构、模板和前端预览，都应逐步向该参考 HTML 对齐。

## 2. 必须吸收的信息架构

参考 HTML 的关键结构：

1. 头部定位
   - 日期、标题、副标题。
   - 用一句话定义当天行情性质，例如“科技内部淘汰赛 · 多方向并行轮动 · 主线换挡日”。

2. 昨日预判验证
   - 昨日核心预测摘要。
   - 今日实际验证结果。
   - 预测正确项、错误/偏保守项、偏差原因。
   - 对后续预判的修正。

3. 先给结论
   - 今日总结判断。
   - 明日核心判断表。
   - 最容易继续、最容易分化、轮动方向、防御方向。

4. 盘面总览
   - 指数数据表。
   - 市场情绪表。
   - 市场结构特征。
   - 资金不是简单流入/流出，而是结构切换。

5. 各板块详细分析
   - 板块状态标题：板块名 + 当日核心判断。
   - 强势/弱势个股表。
   - 板块逻辑分析。
   - 当前阶段判断。
   - 持续性分析。
   - 下个交易日看法。
   - 可以做 / 要回避。

6. 盘后 / 隔夜消息梳理
   - 美股映射。
   - 国内重要消息/催化。
   - 与 A 股板块的映射关系。

7. 板块持续性排序
   - 不是只按当日涨幅，而是按持续性、催化确定性、资金确认度排序。

8. 资金轮动路径分析
   - 今日实际轮动路径。
   - 单极主线是否瓦解。
   - 下一步可能轮动方向。

9. 明日可介入标的与仓位建议
   - 方向、标的、介入方式、仓位、风险。
   - 后续可以保留为“观察清单”，避免输出直接投资建议措辞。

10. 去弱留强排序 / 最实战结论
    - 最值得做的事。
    - 坚决回避的方向。
    - 最终一句核心判断。

## 3. 必须吸收的视觉语言

参考 HTML 的视觉目标是“冷静金融长文”，不是 dashboard。

需要吸收：

- 640px 左右移动端文章宽度。
- 海军蓝 + 金色作为主视觉。
- 米灰背景 + 白色文章卡片。
- `section-num` 小编号块，形成强节奏。
- `conclusion-box` 金色边框结论框。
- `avoid-box` 绿色/风险提示框。
- 表格用于高密度结构化信息。
- 标签区分看多、看空、防御、中性。
- 章节之间使用实线/虚线分隔。
- 整体像公众号/长图，而不是后台管理系统。

## 4. 对当前项目的影响

当前 v0.1 已经有 ReportDTO 和移动端 HTML，但信息结构偏轻，只覆盖：

- 结论
- 盘面总览
- 强势板块
- 个股关注
- 明日观察
- 风险提示

这只能算 MVP，不是目标形态。后续必须升级为 `StructuredReviewDTO` 或扩展 `ReportDTO`，使其能表达：

- 预测验证
- 明日判断表
- 板块详细分析
- 持续性评级
- 资金轮动路径
- 去弱留强排序
- 回避清单
- 消息映射

## 6. 非目标

- 不逐字复制参考 HTML 的具体内容。
- 不直接输出“仓位建议/买卖建议”作为投资指令；可改写为观察清单、风险分层、关注条件。
- 不牺牲事实校验：所有数字、板块、个股必须来自结构化事实或明确人工输入。
- 不把参考 HTML 作为静态模板塞死，必须数据驱动生成。
