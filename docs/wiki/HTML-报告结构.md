# HTML 报告结构

HTML 是本项目最重要的产物。当前渲染风格已经向用户提供的参考 HTML 对齐，使用长文复盘结构，而不是普通后台页面风格。

## 视觉原则

- 宽版文章布局，桌面阅读宽度约 `1080px`。
- 使用统一的卡片、表格、标签、分割线和来源区域。
- 保持中文长文复盘的阅读节奏。
- 重点模块要能直接用于盘后复盘和次日计划。

## 主要模块

当前结构化报告包含：

1. **昨日预判验证**
   - 昨日判断、今日验证、偏差修正。

2. **先给结论**
   - 今日结论、明日核心判断、轮动方向、防御方向。

3. **次日强势概率与观察条件**
   - 板块延续概率。
   - 置信度。
   - 前排股票。
   - 触发条件。
   - 失效条件与风险标签。

4. **盘面总览**
   - 指数数据、涨跌家数、成交额、结构标签。

5. **各板块详细分析**
   - 前排个股。
   - 板块逻辑。
   - 强势/分歧。
   - 次日动作。
   - 持续性分析。

6. **盘后 / 隔夜消息梳理**
   - 海外映射。
   - 国内催化。
   - 风险提示。

7. **板块持续性排序**
   - 高、中、低持续性评级。

8. **资金轮动路径分析**
   - 实际轮动路径。
   - 次日路径观察。

9. **明日可介入标的与仓位建议**
   - 观察候选。
   - 触发逻辑。
   - 仓位纪律。
   - 回避条件。

10. **自选股观察**
    - 默认关闭。
    - 开启后显示自选股强弱、风险和板块匹配。

11. **去弱留强排序**

12. **最实战的结论**

13. **上证指数中期走势研判**

## 仓位建议原则

报告中的仓位建议以纪律表达，不构成投资建议：

- 默认空仓/轻仓观察。
- 只有前排股分歧后重新转强，才考虑试探底仓。
- 单一方向底仓不超过 2 成。
- 主线确认扩散后，总仓位上限控制在 3 成以内。
- 缩量冲高、无催化后排补涨、高位一致加速都属于回避条件。

## 为什么强调前排个股

用户明确要求复盘当天最强势板块和个股，而不是泛泛复盘“垃圾板块”。因此报告必须尽量展示：

- 板块名称。
- 具体股票名称和代码。
- 涨跌幅。
- 来源标签，例如 TickFlow 前排、同花顺复盘、东方财富涨停复盘。
- 次日观察条件。
