# 故障排查

## 1. 报告里出现 fake 或 fallback

症状：

```text
Provider market: fallback
```

处理：

1. 检查 `apps/api/.env`：

```dotenv
MARKET_PROVIDER=tickflow
PROVIDER_FALLBACK_ENABLED=false
TICKFLOW_API_KEY=你的 key
```

2. 重跑：

```bash
make report DATE=YYYY-MM-DD
```

3. 检查 `snapshot.json`：

```bash
jq '.provider_status.market' reports/<DATE>/close/<VERSION>/snapshot.json
```

正式报告应为 `status=success` 且 `fallback_used=false`。

## 2. Anspire 新闻失败

常见原因：

- `ANSPIRE_API_KEY` 未配置。
- API 超时。
- 查询结果为空。

处理：

```dotenv
NEWS_PROVIDER=anspire
ANSPIRE_API_KEY=你的 key
NEWS_TOP_K=10
NEWS_LOOKBACK_HOURS=36
```

## 3. 预览端口打不开

确认你在报告版本目录启动服务：

```bash
cd reports/2026-05-27/close/v003
python3 -m http.server 8890 --bind 127.0.0.1
```

打开：

```text
http://127.0.0.1:8890/report.html
```

如果端口被占用，换一个端口，例如 `8891`。

## 4. Docker credential helper 错误

症状：

```text
error getting credentials - exec: "docker-credential-desktop": executable file not found
```

说明：Docker CLI 配置引用了 Docker Desktop credential helper，但当前 shell 找不到该 helper。

处理建议：

- 打开 Docker Desktop 后重试。
- 或重新执行 `docker login`。
- 或修复 `~/.docker/config.json` 的 credential helper 配置。

## 5. Docker Hub push 权限不足

症状：

```text
insufficient_scope: authorization failed
```

原因：

- Docker Hub 目标仓库不存在。
- 当前 Docker 登录账号没有该仓库写权限。

处理：

1. 创建 Docker Hub 仓库。
2. 确认登录账号。
3. 重新 push。

## 6. HTML 内容过窄或风格不对

当前模板位于：

```text
apps/api/app/renderers/templates/mobile_report.html.j2
```

重点检查：

- `.article-wrap` 最大宽度。
- `.article-card` padding。
- `summary-board`。
- `sector-block`。
- `source-grid`。

报告 UI 应以参考 HTML 风格为准，不再沿用早期应用风格。
