# 每日报告生成流程

本项目的日常入口是：

```bash
make report DATE=YYYY-MM-DD
```

例如：

```bash
make report DATE=2026-05-27
```

## 生成流程

1. 读取 `apps/api/.env` 和环境变量。
2. 通过 TickFlow 获取市场行情和强势板块。
3. 通过 Anspire 搜索板块相关新闻。
4. 抓取同花顺/东方财富复盘信息。
5. 根据行情强度、复盘确认、前排个股、催化和风险生成次日预测。
6. 生成结构化复盘 DTO。
7. 渲染 `report.html`。
8. 导出 `report.png`。
9. 写入 `report_dto.json`、`snapshot.json`、`news_raw.json`、`llm_calls.json`。
10. 写入 SQLite 报告元数据。

## 输出目录

报告默认生成在：

```text
reports/<DATE>/close/vXXX/
```

常见文件：

- `report.html`：最重要的报告产物。
- `report.png`：HTML 截图导出。
- `snapshot.json`：报告、校验、provider 状态的完整快照。
- `report_dto.json`：报告 DTO。
- `news_raw.json`：原始新闻结果。
- `llm_calls.json`：LLM/结构化生成调用记录。

## 如何确认报告没有 fake 内容

打开 `snapshot.json`，检查：

```bash
jq '.provider_status.market' reports/<DATE>/close/<VERSION>/snapshot.json
```

正式报告应看到类似：

```json
{
  "provider": "tickflow",
  "status": "success",
  "fallback_used": false,
  "reason": null
}
```

如果 `fallback_used=true`，不建议把该报告当作正式复盘。

## 清理旧版本

保留最新版本，删除同一天旧版本：

```bash
find reports/2026-05-27/close -maxdepth 1 -type d -name 'v*' ! -name 'v003' -exec rm -rf {} +
```

注意：`reports/` 默认不提交到 GitHub。
